# 개인 개발 환경 설정

## 언어 설정
- 한국어로 응답

## 코딩 스타일
- 명확하고 간결한 코드 작성
- 불필요한 주석 추가하지 말것

## 경로 규칙
- 항상 relative path(상대 경로) 사용
- absolute path 사용 금지
- **파일 읽기/쓰기 시 Bash 도구 사용** (Read/Write 도구는 절대 경로 요구하므로 사용 금지)
  - `cat ~/WorkHistory/...` (읽기)
  - `cat > ~/WorkHistory/... << 'EOF'` (쓰기)

## 워크플로우
- 구현시에 사용자 의도가 명확하지 않은 경우 사용자에게 물어볼것.

## WorkHistory (중앙 집중식 장기 기억) - **CRITICAL**

> **코드 분석, 시스템 탐색, 새로운 정보 발견 시 반드시 즉시 기록!**

- 위치: `~/WorkHistory/`
- 작업 시작 전 `~/WorkHistory/_index.md` 확인
- 프로젝트 구분: `~/WorkHistory/context/projects.md` 참조
- 작성 규칙: `~/WorkHistory/context/writing-guide.md` 참조

### 핵심 규칙
1. **기록 없이 대화 종료 금지** - 분석 결과는 발견 즉시 기록
2. **모든 문서에 핵심 요약(L0) 필수** - 파일 최상단 2-3줄
3. **topics/ 하위에 프로젝트별 저장** - pbb/, pubg/, pubg2/, ctu-example/
4. **task/topic 구분 없음** - 해당 시스템 폴더에 평탄하게 저장

## Cpp

## 자율성 원칙 (필수!)
- **내가 직접 할 수 있는 작업은 절대 사용자에게 요청하지 말것**
- 에디터 실행, 빌드, 파일 생성/수정, MCP 연결 등 도구로 가능한 작업은 스스로 수행
- "~해주세요", "~하면 알려주세요" 같은 요청 금지
- 막히면 다른 방법을 찾아서 직접 해결할것
